# オープン／リバース TCG — ルール仕様書 v0.1（ドラフト）

> このゲームは既存の DCG（リポジトリ直下 `index.html`）とは**完全に別の新しいTCG**です。
> 本書はまず**ルールを曖昧さなく確定する**ための仕様書です。実装はこの確定後に
> `reverse-tcg/index.html` として、既存DCGの盤面描画・通信・PWA基盤を再利用して行います。
>
> 記号の意味:
> - 🟢 **確定** … ユーザー提示／回答で確定済み。
> - 🟡 **提案** … 未定義点をプレイ可能な形で仮置きした案。**レビューで調整可能**。

---

## 1. ゲーム概要 / 勝敗

🟢 2人用の対戦カードゲーム。各カードは**表（オープン面）と裏（リバース面）の両面**を持つ。

### 1.1 勝敗条件 🟢
プレイヤーは次のいずれかを満たした時点で**敗北**する:
1. **ダメージを5つ受ける**（ダメージゾーンのカードが5枚になる）。
2. **デッキアウト** … 山札が0枚の状態でカードを引こうとした。

先に敗北条件を満たしたプレイヤーの相手が勝者。

---

## 2. ゾーン定義 🟢

| ゾーン | 別名 | 説明 | 公開/非公開 |
|---|---|---|---|
| **山札** | デッキ | 引く山。0枚でドロー＝敗北。トリガーチェックもここから捲る。 | 非公開 |
| **手札** | — | 上限5枚（常時処理。超過した瞬間に超過分をボチへ）。 | 自分のみ |
| **盤面** | フィールド | オープンコールしたユニットが並ぶ場所。 | 公開 |
| **ボチ** | 墓地 | 破壊・超過・使用済みカードの置場。 | 公開 |
| **ダメージゾーン** | ダメージ | 受けたダメージのカードが置かれる。5枚で敗北。 | 公開 |
| **オープンカウント** | カウント | オープンコールで溜まる数値リソース。リバースコールで消費。 | 公開（数値） |

> 🟡  オープンカウントは「数値（整数）」として管理する（物理カードを置くのではなくカウンタ）。

## 3. カード構造（両面）🟢/🟡

1枚のカードは**共通情報**と**2つの面**を持つ。

```
┌─────────────────────────────┐
│ 共通: 名前 / トリガーアイコン / 属性    │   ← どちらの面でも変わらない
├──────────────┬──────────────┤
│ オープン面（表）        │ リバース面（裏）       │
│  ・パワー（攻撃/防御値）   │  ・リバースコスト（カウント）  │
│  ・オープン効果        │  ・割り込み効果        │
│  （コスト0で召喚）       │  （相手の行動に割り込む）   │
└──────────────┴──────────────┘
```

- 🟢 **共通情報**: `name`（名前）、`triggerIcon`（トリガーアイコン）、`attr`（属性／色）。
- 🟢 **オープン面**: `power`（パワー）と `openEffect`（オープン時効果）。**コスト0**で盤面に出す。
- 🟢 **リバース面**: `reverseCost`（消費するオープンカウント数）と `reverseEffect`（割り込み効果）。
- 🟡 同じ1枚なので、**手札ではどちらの面でも使える**（オープンコール=表 / リバースコール=裏）。
  使い方を宣言した時点で、その面として処理する。

## 4. ゲーム準備 🟢

1. 各プレイヤーは🟡**40枚程度**のデッキを用意（枚数は提案、レビューで調整可）。
   🟢 **カード/デッキは「山札が1周する」前提でデザインする**（第4.1節）。
2. 山札をシャッフル。
3. **先攻・後攻**を決める（じゃんけん等／実装ではランダム or 選択）。
4. 各プレイヤー**初期手札4枚**を引く。🟡（マリガン有無は未定 → 初版は無しを提案）
5. オープンカウント=0、ダメージゾーン=0枚で開始。

### 4.1 デザイン原則: 「山札が1周する」🟢
🟢 カード効果とデッキ枚数は、**1ゲームで山札がおよそ1周（=デッキ全体を一巡）する**ように設計する。

- 山札はドローだけでなく、**トリガーチェック**（攻撃時・被攻撃時・被ダメージ時）と
  **ダメージゾーン送り**でも消費される。これらを合算して、ゲーム終盤までに山札がほぼ一巡する想定。
- デッキアウトが敗北条件である以上（第1.1節）、ゲーム長は「山札1周」を目安に設計する。
  → トリガーアイコンの配分・各カードのドロー量・デッキ枚数（🟡40枚程度）は、この一巡を基準に調整する。
- 効果で過剰に山札を掘る／回復する設計は避け、**山札残量そのものがリソース・時間制限**として機能する。

## 5. ターン進行 🟢/🟡

各ターンは次のフェイズ順で進む:

| # | フェイズ | 内容 |
|---|---|---|
| 1 | **開始ドロー** | 山札から1枚引く。🟢**先攻1ターン目も引く**。0枚で引けなければ敗北。 |
| 2 | **メイン** | オープンコールでユニットを盤面に出す（後述の配置枚数まで）。 |
| 3 | **攻撃** | 盤面ユニットで攻撃する。🟢**先攻1ターン目は攻撃できない**。 |
| 4 | **エンド** | ターンを相手に渡す。 |

> 🟡 メインと攻撃はDCG同様1つのメインに統合してもよいが、仕様上は分けて記述する。
>
> 🟢 **手札上限はフェイズに依らない常時処理**（第5.2節）。エンドフェイズでまとめて捨てるのではなく、
> いつでも手札が5枚を超えた瞬間に即・超過分をボチへ送る。

### 5.2 手札上限（常時処理）🟢
🟢 手札上限は **5枚**。これは特定タイミングのチェックではなく、**常に働く処理**。
ドロー・トリガー➕・オープン面/リバース面の効果など、**手札が増えたあらゆる瞬間に**判定し、
5枚を超えていれば**超過分を選んで即座にボチへ**送る（自分・相手のどちらのターン中でも同様）。
> リバースコールは相手ターン中の特定タイミングで割り込む（第7章）。

### 5.1 配置枚数（毎ターン1枚ずつ増える）🟢/🟡
🟢「毎ターン、盤面に置けるカードが1枚ずつ増える」。
🟡 これを **「そのターンにオープンコールできる枚数 = 自分のターン数」** と解釈する。

| 自分のターン | オープンコール可能枚数 |
|---|---|
| 1ターン目 | 1枚 |
| 2ターン目 | 2枚 |
| 3ターン目 | 3枚 |
| … | … |

> これは**毎ターンの召喚許容数**であり、盤面の累積上限ではない（前ターンのユニットは残る）。

## 6. オープンコール / オープンカウント 🟢

- **オープンコール** = 手札のカードを**表（オープン面）**で、**コスト0・代償なし**で盤面に出す。
- 出すたびに**オープンカウントが +1**（🟡 標準は+1。トリガー➰でさらに増加し得る）。
- 配置枚数（第5.1節）の上限まで、1ターンに複数回オープンコールできる。
- オープン面の `openEffect` があれば、盤面に出た時に解決する。

> 🟡 オープンカウントの上限は設けない（初版）。バランス次第で上限を後付け可能。

---

## 7. リバースコール 🟢/🟡

- **リバースコール** = **相手の行動に対して**、手札のカードを**裏（リバース面）**で使う割り込み。
- 🟢 コストは**オープンカウントを消費**して支払う（`reverseCost` 数だけ減らす）。
- 使用後のリバースカードは🟡**ボチへ**置く（盤面には残らない＝割り込み専用）。

### 7.1 割り込みできるタイミング 🟡
相手の以下の宣言時に、カウントが足りていれば割り込み可能:
- 相手が**オープンコール**を宣言した時。
- 相手が**攻撃**を宣言した時。

### 7.2 解決順 🟡
- 割り込みは**スタック**（後入れ先出し）で処理する。
- 後から積まれたリバース効果から順に解決し、最後に元の行動を解決する。
- カウント不足ならリバースコールは宣言できない。

> 🟡 リバースコールの応酬（リバースにリバースで返す）を許可するかは初版では「1段まで」を提案。
> 完全なスタック処理は実装第二段階で拡張可能。

---

## 8. トリガーチェック 🟢/🟡

### 8.1 発生タイミング 🟢
次の3つの瞬間に、**山札の一番上を1枚めくって表向きにする**:
1. **攻撃する時**（攻撃側）
2. **攻撃される時**（防御側）
3. **ダメージを受ける時**（ダメージを受ける側）

めくったカードの**トリガーアイコン**が「あり」なら、そのアイコンの効果が発動する。

### 8.2 めくったカードの行き先 🟢/🟡
- 🟢 **「ダメージを受ける時」**のトリガーチェックで捲ったカードは、効果処理の後**ダメージゾーン**へ置く
  （＝ダメージ1つ）。🛡（キャンセル）が出た場合の扱いは下表参照。
- 🟡 **「攻撃する時／される時」**のトリガーチェックで捲ったカードは、効果処理後に**ボチ**へ置く
  （ダメージゾーンには入れない）。

### 8.3 トリガーアイコン一覧 🟡（レビューで確定）

| アイコン | 名称 | 効果 |
|---|---|---|
| ⚔ | 強化 | 戦闘中のユニット1体のパワーを🟡+2する（このターン）。 |
| ➕ | ドロー | カードを1枚引く。 |
| 🛡 | キャンセル | **「ダメージを受ける時」限定で有効**。そのダメージを無効化し、捲ったこのカードはダメージゾーンではなく**ボチ**へ。攻撃/被攻撃時に出た場合は効果なし。 |
| ➰ | カウント | オープンカウントを🟡+1する。 |
| （無印） | なし | 何も起きない。行き先ルール（8.2）に従って置く。 |

> 🟡 アイコンは1カード1種。デッキ内のアイコン配分（例: 各8枚×4種＋無印8枚）はバランス調整で決定。

## 9. 戦闘 🟡（提案モデル）

> ユーザールールに戦闘の詳細記述がないため、矛盾なく遊べる最小モデルを提案する。

### 9.1 攻撃の流れ
1. 攻撃側が盤面のユニット1体で攻撃を宣言。→ **攻撃する時**のトリガーチェック（8.1）。
   このタイミングで相手はリバースコール可能（7.1）。
2. 防御側は盤面のユニット1体で**迎撃（ブロック）**するか、迎撃しないかを選ぶ。
   迎撃宣言時 → **攻撃される時**のトリガーチェック（防御側）。
3. ダメージ決定:
   - **迎撃あり**: 攻撃ユニットと迎撃ユニットの**パワーを比較**。
     - パワーが低い方のユニットを破壊し**ボチ**へ。
     - 同値なら**相打ち**（両方ボチへ）。
     - プレイヤーへのダメージは無し。
   - **迎撃なし**: 防御側プレイヤーが**ダメージを1つ受ける** →
     **ダメージを受ける時**のトリガーチェック（8.1）→ 捲ったカードをダメージゾーンへ（8.2）。

> 🟡 1回の攻撃で与えるプレイヤーダメージは1。複数ダメージを与える強カードは効果で表現。
> 🟡 ユニットの「破壊」は戦闘のみ（パワー比較）。HP/トラフは持たず、パワー1値で判定する簡潔モデル。

## 10. 付録

### 10.1 サンプルカード（両面記載例）🟡

```
[A] 突撃兵
  属性: 赤 / トリガー: ⚔
  ─ オープン面: パワー3 / 効果: なし
  ─ リバース面: コスト2 / 効果: 攻撃されているユニット1体のパワーを-2

[B] 斥候
  属性: 緑 / トリガー: ➕
  ─ オープン面: パワー1 / 効果: 出た時、カードを1枚引く
  ─ リバース面: コスト1 / 効果: 相手のオープンコールに割り込み、そのユニットのオープン効果を無効化

[C] 守護者
  属性: 白 / トリガー: 🛡
  ─ オープン面: パワー2 / 効果: なし
  ─ リバース面: コスト3 / 効果: このターン、自分が受けるダメージを1つ無効化

[D] 増幅器
  属性: 青 / トリガー: ➰
  ─ オープン面: パワー2 / 効果: 出た時、オープンカウント+1
  ─ リバース面: コスト1 / 効果: 攻撃中のユニット1体のパワーを+3（割り込み）
```

### 10.2 プレイ例（1〜2ターンのトレース）🟡

> 曖昧さが残らないことを確認するための進行例。先攻=プレイヤーA、後攻=B。

**ターン1（A・先攻）**
1. 開始ドロー: Aは1枚引く（手札5枚）。🟢先攻でもドローする。
2. メイン: 配置枚数=1。Aは「突撃兵[A]」をオープンコール（コスト0）。盤面に出る。
   オープンカウント 0→1。
3. 攻撃: 🟢先攻1ターン目は攻撃不可。スキップ。
4. エンド: 手札4枚（上限5以下）。ターンをBへ。

**ターン2（B・後攻）**
1. 開始ドロー: Bは1枚引く（手札5枚）。
2. メイン: 配置枚数=1。Bは「守護者[C]」をオープンコール。カウント 0→1。
3. 攻撃: Bは「守護者[C]」(パワー2)で攻撃宣言 →
   **攻撃する時**トリガーチェック: 山札top=「斥候[B]」(➕)→ Bがカード1枚ドロー、捲ったカードはボチへ。
   - ここでAは（カウント1あれば）リバースコール可能。Aはカウント1で「斥候[B]」を裏（コスト1）で
     割り込み… としたいが手札に無ければ不可。今回は割り込まず。
4. 防御: Aは「突撃兵[A]」(パワー3)で迎撃 →
   **攻撃される時**トリガーチェック（A）: top=（無印）→ 効果なし、ボチへ。
5. ダメージ決定: 迎撃あり。パワー比較 3(突撃兵) vs 2(守護者) → 低い守護者を破壊しボチへ。
   プレイヤーダメージ無し。
6. エンド: ターンをAへ。

**（迎撃しない場合の例）** 手順4でAが迎撃しなければ、Aは**ダメージ1**を受ける →
**ダメージを受ける時**トリガーチェック（A）: top=「守護者[C]」(🛡)→ **キャンセル**でこのダメージを無効化、
捲った守護者はボチへ（ダメージゾーンには入らない）。🛡でなければ捲ったカードはダメージゾーンへ置かれ、
ダメージゾーンが5枚になればAの敗北。
